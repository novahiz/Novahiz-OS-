# Workflow: Project Kickoff

> Standard protocol for starting a new project.

## Trigger
- New project brief received
- User wants to build something new
- `new project`, `kickoff`, `start`

## Steps

### 1. Discovery (Victor + Elias)
- [ ] What is the project? (1-paragraph description)
- [ ] Who are the users?
- [ ] What does success look like? (measurable)
- [ ] Deadline / milestones?
- [ ] Budget / constraints?

### 2. Tech Scoping (Arthur + Ralph)
- [ ] Architecture recommendation
- [ ] Stack decisions
- [ ] Integration points
- [ ] MVP scope definition

### 3. Agent Assignment (Victor + Victor)
- [ ] Assign lead agent
- [ ] Assign support agents
- [ ] Define responsibilities

### 4. Project Setup (Ralph)
- [ ] Create `02_Projects/<name>/`
- [ ] Fill `_template.md` with discovered context
- [ ] Initialize nexus.json
- [ ] Set current project in `05_Context/current.md`

### 5. First Sprint Planning (Victor)
- [ ] Define sprint 1 goals (1-2 weeks max)
- [ ] Create first tasks in project context
- [ ] Set milestone with due date

### 6. Kickoff Meeting
- [ ] Brief council deliberation (MEDIUM — 3 sages)
- [ ] Document first decision: architecture + stack
- [ ] Record in ADR if significant
