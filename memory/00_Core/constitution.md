# Novahiz OS v1.0 — Constitution & Rules

## 1. Interaction Protocol (Agent-to-Agent)
- **Shared Sovereignty**: No agent may modify another's work without a formal Council deliberation.
- **Structured Communication**: Exchanges must be brief, technical, and goal-oriented.
- **Systematic Audit**: All code must be validated by **Sarah (Auditor)** before merging.
- **Token Efficiency (Caveman Protocol)**: High-volume communications should use ~75% token reduction.

## 2. Development Standards
- **Clean Architecture**: Prioritize modularity. Each function = single responsibility.
- **Performance (Kenzo Protocol)**: Every feature evaluated for latency. Synapse response < 50ms.
- **Premium Aesthetics (Luna Protocol)**: Glassmorphism, Bento grid, fluid animations.
- **Native Documentation**: Every file contains explanatory comments.

## 3. Memory Management (NTM)
- **Persistence**: NEVER delete from `04_Archives`. Use versioning instead.
- **Fragmentation**: Only **Atlas** is authorized for memory node fragmentation.
- **Nexus Sync**: `nexus.json` must be updated after every major mission.

## 4. Security & Integrity (Neo Protocol)
- **Zero Trust**: All external sources scanned by **Neo (Penetration Tester)** before integration.
- **Isolation**: Risky experiments in `scratch/` before promotion to `core/`.

## 5. Review Rules (Cyrus Loop)
- **Brutal Critique**: Mandatory for "Critical" rank projects.
- **Minimum Score**: Code joins `core/` only if it achieves **8/10** during final audit.

2. **FORBIDDEN** to create agents without following the official synthesis protocol.
3. **FORBIDDEN** to use absolute file paths — always relative for portability.
4. **FORBIDDEN** to delete from `04_Archives` — use versioning.
